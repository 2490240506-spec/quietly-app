# Quietly 产品说明

## 1. 产品名称

Quietly

## 2. 一句话介绍

Quietly 是一个移动端优先的 AI 声音掩蔽工具。它通过本地分析环境声，识别主要噪音类型，并自动匹配适合的专注声场，帮助用户在办公室、图书馆、自习室等场景降低干扰感。

## 3. 产品定位

Quietly 不是主动降噪耳机，也不是录音识别工具。它的核心是“环境声分析 + 声音掩蔽推荐”：根据当前环境的声音特征，选择更适合的背景声场，让用户在嘈杂但无法改变的环境中更容易专注、放松或入眠。

第一版优先做成浏览器即开即用的 Web MVP，后续可迁移到微信小程序。

## 4. 目标用户

- 在办公室被聊天声、键盘声、空调声干扰的人
- 在图书馆、自习室、咖啡馆学习但容易被环境声打断的人
- 睡前需要舒缓背景声辅助放松的人
- 不想下载 App，希望打开链接即可使用的轻量用户

## 5. 核心问题

很多用户并不需要完全消除噪音，而是需要降低噪音的突兀感和注意力占用。传统白噪音工具通常只提供固定播放列表，用户需要自己判断播放什么声音。Quietly 希望把这个判断过程变得更自动、更贴近场景。

## 6. MVP 核心功能

### 本地环境声分析

用户授权麦克风后，Quietly 使用浏览器 Web Audio API 获取环境声的实时频谱和音量特征。当前版本只做本地分析，不录音、不保存、不上传。

### 噪音粗分类

MVP 使用规则引擎判断主要环境状态，包括：

- 安静环境
- 高频干扰
- 人声聊天
- 低频轰鸣
- 稳定背景噪音

这不是训练后的大模型识别，而是基于音量、频段能量、波动特征做的轻量分类，适合比赛 Demo 和产品原型验证。

### 声场推荐

| 环境判断 | 推荐声场 | 设计目的 |
| --- | --- | --- |
| 高频干扰 | 雨声专注 | 用细密雨声覆盖尖锐、断续的高频噪音 |
| 人声聊天 | 咖啡馆低语 | 用稳定人声氛围降低单个对话的突兀感 |
| 低频轰鸣 | 海洋低频 | 用连续低频波浪感中和空调、车辆等低频干扰 |
| 安静环境 | 森林轻风 | 保持轻量自然背景，避免过度刺激 |
| 稳定噪音 | 深度专注 | 用更稳定的底噪帮助进入长时间专注状态 |

### 五个声场

- 雨声专注
- 咖啡馆低语
- 海洋低频
- 森林轻风
- 深度专注

每个声场都有对应插图、动态层、颜色氛围和音频文件。

## 7. 视觉与交互方向

Quietly 的视觉目标是“治愈、安静、像小程序而不是网页”。界面以手机竖屏为主，强调第一屏的沉浸式场景：

- 首屏 Hero 舞台展示当前声场插图
- 声场切换时背景色、插图、天气层、标签和声波状态同步变化
- 场景内有轻微动效，例如窗帘摆动、雨滴、海浪、漂浮光点
- 下滑浏览声场卡片时有平滑入场动画
- 整体避免营销页风格，重点保持工具感和可演示性

## 8. 隐私设计

Quietly 的隐私原则：

- 只做本地实时分析
- 不录制用户声音
- 不保存音频文件
- 不上传麦克风数据
- 用户可以随时关闭环境分析

这对办公室、图书馆和自习室场景很重要，因为这些环境可能包含他人对话。

## 9. 技术实现

当前 MVP 技术栈：

- React
- TypeScript
- Vite
- Web Audio API
- CSS Modules
- 本地静态音频与图片资源

核心流程：

1. 用户点击开始
2. 浏览器请求麦克风权限
3. Web Audio API 分析实时频谱
4. 规则引擎判断环境噪音类型
5. 推荐最适合的声场
6. 前端切换视觉场景并播放对应声音

## 10. 当前 MVP 边界

当前版本适合作为比赛 Demo 和产品验证原型，但仍有明确边界：

- 不是医学或治疗产品
- 不是主动降噪技术
- 当前分类为轻量规则判断，不是深度学习模型
- 对极端复杂环境的识别可能不稳定
- 小程序版本需要后续使用微信原生能力重新适配麦克风、音频播放和权限流程

## 11. 后续计划

后续可以分三步升级：

1. 优化噪音识别稳定性：增加更长时间窗口、置信度判断和切换冷却机制。
2. 增强个性化：允许用户标记“这个声场有效 / 无效”，逐步形成个人推荐偏好。
3. 迁移微信小程序：使用 Taro 或微信原生框架重构界面，并接入微信录音与音频播放 API。
